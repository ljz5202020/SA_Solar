### GT 加热器污染审计

**背景**: Codex 提出 GT 中可能混入太阳能热水器/泳池加热器，cross-review 后发现两个关键问题需修正：
1. 94 个 cleaned GPKG 有两种 schema（早期 17 文件 vs Apr3 77 文件），Codex 假设的 `annotation_id` 主键对 93% 数据不成立
2. 风险分层依赖的 `num_parts` 字段在大多数文件中缺失，改用 geometry_bucket + solidity

**实现**: 3 个脚本
- `build_gt_heater_audit.py` — 扫描 cleaned/ 构建审计队列 + 导出 400×400 chip
- `label_gt_heater_audit.py` — 自包含 HTML 标注器（1=PV, 2=热水器, 3=不确定）
- `export_coco_dataset.py` 新增 `--audit-csv` 过滤参数

**Phase 1 结果** (tier A: compact, solidity>0.9, 5-30m²):

| 指标 | 值 |
|------|-----|
| 总 GT 扫描 | 1731 条 (20 grids) |
| Tier A 队列 | 855 条 |
| 可标注 (有 chip) | 671 条 (184 条 G1688 buffer 区跳过) |
| 标注结果 | **585 PV / 80 heater / 6 uncertain** |
| **污染率** | **12.8%** |

热水器特征: area 5.1-29.2m² (中位 14.7), 全部来自 260403 batch, G1687(30) 和 G1686(17) 污染最重。

**结论**: batch 003 及之前标注时对泳池加热器认知不足，导致 80 条热水器混入 GT。已生成 canonical 过滤 CSV，下次训练导出时自动排除。
